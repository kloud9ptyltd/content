The idea — how it holds together

1 Built in Claude Code

The whole desk is built and driven with Claude Code. You describe an agent; it writes it. This is what turns "an AI that talks" into a team that does the work.

Highly recommended

Run it on Claude's Fable 5 model. An orchestrator that routes one spoken command to the right specialists — and knows when to stop and let the risk officer say no — lives or dies on reasoning quality. Fable 5 is the difference between a real desk and nine chatbots talking over each other. If you're serious, build on Fable 5. Why Fable 5 for finance →

2 One master, many specialists

Each agent is its own independent service with a single job and a clean way to talk to the others. Nimbus is the only one that talks to you — it delegates down and synthesizes up. Adding a tenth agent later doesn't touch the other nine.

3 The voice — talk in, talk back

Speech-to-text in the browser lets you just speak. For the replies, ElevenLabs' API gives Nimbus and every agent a distinct voice — so the roll call feels like a real team, not one robotic monotone.

You'll need an ElevenLabs API key and an Anthropic API key (bring your own).

4 The one rule that makes it safe

Money moves through exactly one gated path: a proposal only becomes a trade after Sentinel stamps it. Analysis can't execute; execution can't self-authorize. The risk officer is built into the architecture — not bolted on.

⚠️ What it is — and isn't: Nimbus is a research and awareness co-pilot with a built-in risk officer, not a money printer. It surfaces structure and context; it does not tell you what will happen next. Educational, grounded, and not financial advice — always do your own research.
